🔁 3-Layer Hybrid Detection Pipeline

Layer 1 — Allowlist
150+ globally trusted domains (Google, YouTube, SBI, HDFC, Yenepoya, etc.) are instantly marked CLEAN (2% risk) — no ML needed. Prevents false positives on famous sites.
Layer 2 — Rule Blocklist
Structural red flags (raw IP host, @ symbol, punycode/homograph, brand in subdomain, suspicious TLD + login keywords) → instantly HIGH RISK (97%) regardless of ML.
Layer 3 — ML Model
URLs that pass layers 1 & 2 go through the Random Forest. 21 features are extracted, scaled, and fed to 200 trees. Final score = average probability across all trees (0–100%).

Why Random Forest? — Justification

Handles non-linear relationships between URL features without manual feature engineering
Resistant to overfitting — ensemble of 200 trees, each trained on a random feature subset
No assumptions about data distribution — works well on mixed numeric features
Naturally provides probability scores (predict_proba) for calibrated risk %
Fast inference — sub-millisecond prediction per URL, suitable for real-time browser extension
Built-in class_weight="balanced" handles unequal phishing vs. legit dataset sizes

⚖️ Why Trust This Model? — Honest Assessment
Strengths, limitations, and how we address them
✅ Strengths
Works entirely on URL structure — no need to visit the site
3-layer hybrid approach reduces both false positives and false negatives
Explainable — every decision shows which features triggered it
Real-time — sub-ms prediction, integrated into browser extension
Open-source dataset (phishing_dataset.csv) — reproducible results
⚠️ Known Limitations
Lexical-only — cannot detect phishing on legitimate domains (e.g., compromised pages)
Dataset may not cover very new phishing patterns (zero-day campaigns)
WHOIS data unavailable for some new/private domains
URL shorteners hide the real destination — flagged as suspicious but cannot resolve
